Sobre Aux

control_arg:

Parâmetro Padrão | Opcional

Descrição:

Trata-se de um parâmetro opcional padrão presente em quase todos os módulos AUX. Um módulo pode ter ou não um ou mais parâmetros denomidados de control_arg (control_arg2, control_arg3...) que são usados para alterar o comportamento padrão de um módulo, isso redefine o modo como ele trabalha e retorna um resultado.

Um módulo tem sua funcionalidada padrão preservada quando os parâmetros control_arg são omitidos. Mas se um argumento válido for passado sua funcionalidade é definida para o que o argumento solicita, isso também pode mudar os valores e tipo que esse módulo retorna. Estes argumentos válidos são os chamados de argumentos de controle e são geralmente uma string passada no parâmetro e esses valores string são geralmente pré-definidos e cada módulo tem seus argumentos padrões. Para saber quais argumentos um módulo recebe deve-se verificar a documentação deste. Se o módulo possuir um ou mais parâmetros de argumentos de controle a documentação listará todos os argumentos que cada parâmetro pode receber, o que ela faz e suas regras.

Alguns parâmetros possuem regras e só podem ser usados em determinados momentos. Caso essa regra não seja seguida um erro é disparado interrompendo a execução do código. Veja a seguir:

Usando argumentos de controle:

Temos uma função chamada arrMatch( ) que faz uma verificação em dois arrays e retorna true se um ou mais elementos do primeiro array também existir no segundo array. Este é o comportamento padrão da função se apenas os arrays forem passados como argumentos.




                            
Mas esta função possui mais dois parâmetros opcionais; os control_arg. Lendo a documentação sobre esta função descobrimos que o terceiro parâmetro recebe alguns argumentos do tipo string pré-definidos. São eles:

                            
    
                                
  • "all"
    • 
                                
  • "list"
    • 
                                
  • "details"
    • 
                                
  • "length"
    • 
                                
  • "default"
    • 
                            


                            
Nota: Os argumentos listados e outros serão usados como exemplos. A funcionalidade de cada argumento apresentado abaixo não é uma regra para todos os módulos que as usam, onde cada módulo pode retornar resultados diferentes usando os mesmos argumentos desta lista.


                            
Vamos ver o que cada um faz.

                            
O argumento "all" se declarado em control_arg diz para a função verificar se todos os valores de arrA existem em arrB. O retorno ainda será um boolean.

                            


                            
Já o argumento "list" altera tanto o modo de verificação quanto o valor de retorno. Se usado diz para a função verificar quais valores do primeiro array estão presentes no segundo array e retorna um novo array contendo esses valores.

                            
O argumento "details" trabalha semelhante ao argumento "list" porém retorna um array contendo um objeto para cada item de arraA que estiver presente em arrB. Cada objeto tem propriedades que fornece algumas informações sobre a busca.

                            
O argumento "length" retorna um número que representa a quantidade de valores de arrA que estão presentes em arrB.

                            
Já o último argumento "default", presente em todos os módulos que possuem um parâmetro control_arg, é o argumento padrão deste parâmetro. Se nada for passado em nenhum parâmetro de argumento de controle o valor padrão é "default".

                            

                            
Note que cada argumento tem a habilidade que mudar toda a estrutura do retorno da função deste exemplo, isto sempre deve ser levado em consideração na hora de usá-los em um projeto, pois o não conhecimento destes pode ababar trazendo resultados inesperados que podem comprometer a estrutura do projeto.


                            
                        

                        
                        

                            
control_arg2 ?:


                            
Uma função pode possuir mais de um parâmetro do tipo control_arg, se você se notar isso em um módulo isso significa a funcionalidade do argumento anterior pode trabalhar em conjunto com mais funcionalidades. Em outras palavras um segundo parâmetro do tipo control_arg altera a funcionalidade do primeiro parâmetro do mesmo tipo.

                            
Parece confuso? Calma que há como explicar.

                            
Vamos usar a mesma função dos exemplos acima, já que este também possui um quarto parâmetro chamado control_arg2. Além de argumentos pré-definidos do tipo string, o parâmetro anterior, control_arg também aceita receber um número como argumento, se passado um argumento numérico inteiro a função irá verificar se a quantidade de valores de arrA que estão presentes em arrB é igual ao número passado e retorna um boolean. Com isto podemos usar mais dois argumentos pre-definidos no parâmetro control_arg2. São eles:


                            
    
                                
  • "min"
    • 
                                
  • "max"
    • 
                            


                            
O argumento "min" define que a quantidade mínima de valores de arrA que correspondem deve ser o valor passado no parâmetro anterior e retorna true, se a quantidade passar o retorno é false, enquanto o argumento "max" faz o oposto e define que a quantidade máxima de valores de arrA que correspondem à arrB deve ser o valor passado anteriormente e também retorna um boolean. Veja:

                            

                            
Note que o uso de um segundo parâmetro do tipo control_ar dita novas regras sobre as regras já estabelecidades no uso do parâmetro anterior. Deve-se notar também parâmetros dete tipo podem receber outros tipos de argumentos além dos padrões pre-defindios, como é o caso desta função.

                            
Os argumentos listados aqui não são todos. Cada módulo possui sua particularidade e pode ou não compartilhar dos mesmos argumentos, ou podem ou não usar os mesmos argumentos em situações diferentes com resultados diferentes.
